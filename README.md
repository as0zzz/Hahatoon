# TTM — Система управления задачами «Транстелематика»

Корпоративное Django-приложение для компании «Транстелематика»: управление задачами, планирование, оценка рисков, дашборды, роли, AI-помощник (Gemini) и генерация отчётов.

---

## Быстрый старт (Windows)

### 1. Клонировать репозиторий

```powershell
git clone https://github.com/as0zzz/Hahatoon.git
cd Hahatoon
```

### 2. Создать виртуальное окружение и активировать его

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

### 3. Установить зависимости

```powershell
pip install -r requirements.txt
```

### 4. Настроить базу данных PostgreSQL

Проект использует **PostgreSQL** как основную базу данных.

#### 4.1. Убедитесь что PostgreSQL установлен и запущен

Если PostgreSQL ещё не установлен, скачайте с [postgresql.org](https://www.postgresql.org/download/windows/).

#### 4.2. Создайте базу данных

Откройте **pgAdmin** или **psql** и выполните:

```sql
CREATE DATABASE ttm;
```

> Если пользователь/пароль PostgreSQL отличаются от `postgres`/`postgres`, отредактируйте файл `.env`:
>
> ```
> POSTGRES_USER=ваш_пользователь
> POSTGRES_PASSWORD=ваш_пароль
> ```

#### 4.3. Применить миграции и загрузить демо-данные

```powershell
python manage.py migrate
python manage.py seed_data
```

Команда `seed_data` создаёт: 7 отделов, 7 команд, 10 пользователей, 36 задач, комментарии, историю, уведомления и отчёт.

### 5. Запустить сервер

```powershell
python manage.py runserver
```

Откройте в браузере: **http://127.0.0.1:8000/**

## Конфигурация (.env)

Файл `.env` уже включён в репозиторий с рабочими настройками. Дополнительная настройка **не требуется**, если PostgreSQL работает на стандартных параметрах (`localhost:5432`, пользователь `postgres`, пароль `postgres`).

Ключевые переменные:

| Переменная          | Значение по умолчанию                       | Описание                                    |
|---------------------|---------------------------------------------|---------------------------------------------|
| `SECRET_KEY`        | `django-insecure-ttm-demo-only-...`         | Ключ Django (для демо)                      |
| `DEBUG`             | `true`                                      | Режим отладки                               |
| `DATABASE_URL`      | `postgresql://postgres:postgres@localhost:5432/ttm` | Подключение к PostgreSQL              |
| `POSTGRES_DB`       | `ttm`                                       | Имя базы данных                             |
| `POSTGRES_USER`     | `postgres`                                  | Пользователь PostgreSQL                     |
| `POSTGRES_PASSWORD` | `postgres`                                  | Пароль PostgreSQL                           |
| `GEMINI_API_KEY`    | *(ключ включён)*                            | API-ключ для Google Gemini AI               |
| `AI_MOCK_MODE`      | `false`                                     | `false` — реальный AI, `true` — заглушка    |

## Основные функции

- **Канбан-доска задач** — управление задачами с drag-and-drop, фильтрация и поиск
- **Иерархия задач** — годовые → квартальные → месячные → недельные цели
- **Автоматическая оценка рисков** — на основе дедлайнов, прогресса и нагрузки
- **AI-помощник (Gemini)** — анализ задач, рекомендации, генерация описаний
- **Дашборды** — статистика, графики, сводки по подразделениям
- **Отчёты** — генерация и экспорт Excel отчётов
- **Уведомления** — оповещения о рисках, дедлайнах и изменениях
- **Журнал аудита** — полная история всех действий
- **Ролевая модель** — сотрудник, руководитель команды, руководитель подразделения, администратор
