# Архитектура TTM

## Стек

- Django 5.2 и PostgreSQL через переменные окружения.
- Django Templates и статические CSS/JS без обязательного Node.js.
- Chart.js подключён для визуализации дашбордов.
- AI вынесен в `apps/ai_assistant/services.py`: mock-режим работает без ключа, Gemini подключается через `GEMINI_API_KEY`.

## Приложения

- `apps/accounts` — профиль пользователя, роли и настройки.
- `apps/departments` — отделы, команды, страницы сотрудников и отделов.
- `apps/tasks` — задачи, комментарии, история, формы, CRUD, планирование, канбан, календарь, расчёт риска и seed-команда.
- `apps/dashboards` — главная страница и аналитические дашборды.
- `apps/ai_assistant` — AI-помощник, отчёт, извлечение задачи из текста.
- `apps/reports` — отчёты и Excel-экспорт.
- `apps/notifications` — уведомления пользователя.
- `apps/audit` — журнал действий.

## Поток данных

Задачи хранят статус, период планирования, дедлайн, приоритет, прогресс и связи с отделом/ответственным. При создании, редактировании, смене статуса и построении дашбордов вызывается сервис риска. Дашборды получают агрегаты через `get_task_metrics`, AI использует только агрегаты и ограниченный срез задач.

## Суверенность

Приложение запускается локально без Docker и без обязательных внешних SaaS. AI может работать в локальном fallback-режиме, поэтому корпоративные данные не передаются наружу без явной настройки провайдера.
