# Поэтапный план разработки TTM

**Продукт:** TTM  
**Компания:** Транстелематика  
**Формат документа:** пошаговая инструкция для нейросети-разработчика  
**Основа:** спецификация `TTM_SPEC.md`  
**Основной стек:** Django  
**AI-модель:** Gemini 3.1 Flash Lite  
**Ограничение:** Docker не использовать.

---

## 0. Главное правило работы

Не пытайся создать весь проект одним большим ответом или одним большим изменением. Разрабатывай TTM маленькими проверяемыми шагами.

После каждого шага обязательно:

1. Проверь, что изменённые файлы сохранены.
2. Запусти проверки в терминале.
3. Посмотри ошибки полностью, а не только последнюю строку.
4. Исправь ошибки до перехода к следующему шагу.
5. Если ошибка непонятна, сначала локализуй её: какой файл, какая строка, какая команда падает.
6. Не переходи дальше, пока текущий шаг не работает.

Базовые команды проверки, которые нужно использовать регулярно:

```bash
python manage.py check
python manage.py test
python -m compileall .
```

Если на текущем этапе ещё нет `manage.py`, используй:

```bash
python --version
pip --version
```

Если после изменения моделей появились новые поля или модели:

```bash
python manage.py makemigrations
python manage.py migrate
python manage.py check
python manage.py test
```

Если изменялись шаблоны, URL или views:

```bash
python manage.py check
python manage.py runserver
```

После запуска сервера вручную открой ключевые страницы в браузере и проверь, что нет ошибки 500, не сломана вёрстка, нет пустой страницы без причины.

## 1. Рабочий протокол для нейросети

При выполнении каждого шага придерживайся формата:

```md
## Текущий шаг: <номер и название>

Что делаю:
- ...

Какие файлы меняю:
- ...

Команды проверки:
```bash
...
```

Результат проверки:
- успешно / есть ошибка;
- если есть ошибка — что исправлено.
```

Не делай одновременно изменения из разных больших областей. Например, не создавай сразу модели, дашборды, AI и отчёты в одном шаге. Сначала модель, потом миграция, потом админка, потом seed-данные, потом страница.

---

## 2. Definition of Done для каждого шага

Шаг считается завершённым только если:

- код запускается без синтаксических ошибок;
- `python manage.py check` проходит успешно;
- миграции применены, если они нужны;
- страница открывается, если шаг связан с UI;
- форма сохраняет данные, если шаг связан с формой;
- фильтр действительно фильтрует, если шаг связан с фильтрами;
- тестовые данные не ломаются;
- нет заглушек вида `TODO`, которые блокируют сценарий демонстрации;
- интерфейс остаётся на русском языке;
- дизайн остаётся в стиле Транстелематики.

**Цель:** подготовить чистую директорию для проекта.

**Действия:**

- Создай папку `ttm`.
- Перейди в неё.
- Убедись, что внутри нет лишних файлов от других проектов.

**Проверка:**

```bash
pwd
ls -la
```

**Готово, если:** папка открыта, структура пустая или содержит только исходные служебные файлы проекта.

**Цель:** изолировать зависимости проекта.

**Действия:**

```bash

```

Активируй окружение.

Linux / macOS:

```

```

# Этап 23. Что делать при ошибках

## Ошибка миграций

1. Прочитай traceback полностью.
2. Найди модель и поле, которые вызвали ошибку.
3. Не удаляй миграции вслепую.
4. Если проект ещё на раннем этапе и данных нет, допустимо пересоздать БД и миграции, но только осознанно.
5. После исправления выполни:

```bash
python manage.py makemigrations
python manage.py migrate
python manage.py check
```

---

## Ошибка 500 на странице

1. Посмотри traceback в терминале.
2. Найди файл шаблона или view.
3. Проверь, какие переменные передаются в контекст.
4. Если ошибка в шаблоне, временно выведи минимальный контент.
5. После исправления открой страницу снова.

---

## Ошибка JavaScript

1. Открой консоль браузера.
2. Найди файл и строку.
3. Проверь, существуют ли DOM-элементы, к которым обращается JS.
4. JS не должен ломать всю страницу, если элемента нет.
5. Добавь проверки `if (element) { ... }`.

---

## Ошибка AI

1. Приложение не должно падать из-за AI.
2. Если нет ключа Gemini, используй fallback.
3. Если API вернул ошибку, покажи пользователю аккуратное сообщение.
4. Не блокируй создание задач и просмотр дашбордов.

---

## Ошибка прав доступа

1. Проверь текущего пользователя.
2. Проверь его `EmployeeProfile`.
3. Проверь роль.
4. Проверь queryset.
5. Проверь прямой URL, а не только скрытие кнопки.

# Финальная инструкция для нейросети-разработчика

Разрабатывай TTM итерационно. Выполняй один шаг за раз. После каждого шага запускай проверки, смотри терминал, исправляй ошибки и только потом продолжай. Не скрывай ошибки и не оставляй сломанные страницы «на потом». Главная цель — получить стабильный демонстрационный продукт, который выглядит как корпоративная система управления задачами, а не как набор несвязанных заготовок.

Проект должен запускаться без Docker, использовать Django как основу, иметь русский интерфейс, фирменный стиль Транстелематики, демо-данные, роли, задачи, дашборды, риски и AI-помощника на Gemini 3.1 Flash Lite с безопасным fallback без API-ключа.
